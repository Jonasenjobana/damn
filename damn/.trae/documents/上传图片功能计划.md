# 上传图片功能计划

## 需求分析
1. 提供 `/api/upload/image` 接口上传图片
2. 图片存储在项目根目录 `resource/` 文件夹
3. 设计数据库表记录图片信息

## 实现步骤

### 1. 设计数据库表结构

#### upload_file（文件上传表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键 |
| original_name | VARCHAR(255) | 原始文件名 |
| filename | VARCHAR(255) | 存储文件名（UUID） |
| path | VARCHAR(500) | 文件路径 |
| mime_type | VARCHAR(100) | 文件类型 |
| size | BIGINT | 文件大小（字节） |
| extension | VARCHAR(20) | 文件扩展名 |
| type | TINYINT | 文件类型 1-图片 2-文件 |
| status | TINYINT | 状态 1-正常 0-禁用 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| is_deleted | TINYINT | 是否删除 |

## 接口设计

### 上传图片接口
- **路径**: `POST /api/upload/image`
- **认证**: 需要登录
- **请求**: multipart/form-data
  - field: `file` (图片文件)
- **响应**: 
```json
{
  "rlt": "0",
  "msg": "上传成功",
  "data": {
    "id": 1,
    "filename": "uuid-xxx.jpg",
    "path": "/resource/images/uuid-xxx.jpg",
    "url": "/api/upload/images/uuid-xxx.jpg"
  }
}
```

### 获取图片接口
- **路径**: `GET /api/upload/images/:filename`
- **响应**: 图片文件

### 删除图片接口
- **路径**: `DELETE /api/upload/images/:id`
- **认证**: 需要登录
- **响应**: 
```json
{
  "rlt": "0",
  "msg": "删除成功"
}
```

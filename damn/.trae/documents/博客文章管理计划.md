# 博客文章管理计划

## 需求分析
1. 文章表 - 存储文章信息（MD格式内容、封面、标题、私密、排序、pin标记）
2. 文章类型表 - 管理文章分类
3. 文章与类型关联 - 创建文章时选择类型

## 实现步骤

### 1. 设计数据库表结构

#### article_type（文章类型表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键 |
| name | VARCHAR(50) | 类型名称 |
| sort | INT | 排序 |
| status | TINYINT | 状态 1-正常 0-禁用 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| is_deleted | TINYINT | 是否删除 |

#### article（文章表）
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键 |
| title | VARCHAR(200) | 文章标题 |
| content | TEXT | 文章内容（MD格式） |
| cover | VARCHAR(500) | 封面URL |
| article_type_id | BIGINT | 文章类型ID（逻辑关联） |
| sort | INT | 排序 |
| is_pinned | TINYINT | 是否置顶 1-是 0-否 |
| is_private | TINYINT | 是否私密 1-是 0-否 |
| status | TINYINT | 状态 1-正常 0-禁用 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| is_deleted | TINYINT | 是否删除 |

## 接口设计

### 文章类型接口
- `GET /api/article-type/list` - 获取类型列表
- `POST /api/article-type/create` - 创建类型
- `PUT /api/article-type/update/:id` - 更新类型
- `DELETE /api/article-type/delete/:id` - 删除类型

### 文章接口
- `GET /api/article/list` - 获取文章列表（支持分页、筛选）
- `GET /api/article/:id` - 获取文章详情
- `POST /api/article/create` - 创建文章
- `PUT /api/article/update/:id` - 更新文章
- `DELETE /api/article/delete/:id` - 删除文章
- `PUT /api/article/pin/:id` - 置顶/取消置顶
- `PUT /api/article/sort` - 批量排序

## 注意事项
- 严格遵循 db-rule.md 数据库设计规范
- 所有接口使用统一响应格式
- 敏感接口添加 JWT 认证
- 文章列表优先按 is_pinned 排序，再按 sort 排序

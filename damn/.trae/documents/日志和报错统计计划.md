# 日志和报错统计计划

## 目标
1. 为所有接口添加统一的日志记录
2. 对报错接口进行统计
3. 实现统一的 RESTful 响应格式
4. 添加接口前缀 `/api/`

## 实现步骤

### 1. 创建统一响应格式
- 创建 `common/response.ts` 定义统一的响应结构
```typescript
{
  rlt: "0",  // 0 成功 1 失败
  msg: "success",  // 默认success 自定义失败信息
  data: {}  // 成功时返回的数据 可以是数组
}
```

### 2. 创建日志服务
- 创建 `common/logger.service.ts`
- 记录接口请求日志（方法、路径、参数、响应时间）
- 记录错误日志（错误信息、堆栈、接口信息）
- 统计报错接口（按接口路径、错误类型统计）

### 3. 创建日志拦截器
- 创建 `common/logging.interceptor.ts`
- 拦截所有请求
- 记录请求和响应
- 统计接口调用次数和报错次数

### 4. 创建异常过滤器
- 创建 `common/http-exception.filter.ts`
- 捕获所有异常
- 统一错误响应格式
- 记录错误日志

### 5. 创建接口前缀
- 在 `main.ts` 中设置全局前缀 `/api`

### 6. 改造现有 Controller
- 更新 `AuthController` 使用统一响应格式
- 更新 `UserController` 使用统一响应格式
- 确保所有接口都返回 `{ rlt, msg, data }` 格式

### 7. 创建报错统计服务
- 创建 `common/error-stats.service.ts`
- 内存存储报错统计
- 提供查询统计信息的接口

## 文件结构
```
src/
├── common/
│   ├── response.ts              # 统一响应格式
│   ├── logger.service.ts        # 日志服务
│   ├── logging.interceptor.ts   # 日志拦截器
│   ├── http-exception.filter.ts # 异常过滤器
│   └── error-stats.service.ts   # 报错统计服务
├── modules/
│   ├── auth/
│   │   └── auth.controller.ts   # 更新使用统一响应
│   └── user/
│       └── user.controller.ts   # 更新使用统一响应
└── main.ts                      # 设置全局前缀、注册拦截器和过滤器
```

## 功能特性

### 日志记录
- 请求日志：方法、路径、参数、响应时间、状态码
- 错误日志：错误信息、堆栈、接口路径、请求参数
- 统计信息：每个接口的调用次数、成功次数、失败次数

### 报错统计
- 按接口路径统计报错次数
- 按错误类型统计
- 按时间范围查询统计

### 统一响应
- 成功：`{ rlt: "0", msg: "success", data: ... }`
- 失败：`{ rlt: "1", msg: "错误信息", data: null }`

# 文章统计功能实现计划

## 功能概述

为博客系统添加以下功能：
1. 文章访问人数统计（防刷新）
2. 文章点赞功能
3. 累积浏览时长统计
4. 访客 Cookie 指纹（1小时过期）

## 数据库设计

### 1. 更新 Article 实体

在 `article.entity.ts` 中添加以下字段：
- `view_count` - 浏览次数（int, default: 0）
- `like_count` - 点赞数（int, default: 0）
- `total_view_duration` - 总浏览时长（秒, int, default: 0）

### 2. 新建 ArticleView 实体

用于记录访客浏览记录（防止短时间内重复计数）：
- `id` - 主键
- `article_id` - 文章ID
- `fingerprint` - 访客指纹（Cookie）
- `view_start_time` - 浏览开始时间
- `view_duration` - 本次浏览时长（秒）
- `create_time` - 创建时间

### 3. 新建 ArticleLike 实体

用于记录点赞记录：
- `id` - 主键
- `article_id` - 文章ID
- `fingerprint` - 访客指纹（未登录用户）或 `user_id`（登录用户）
- `user_id` - 用户ID（可选）
- `create_time` - 创建时间

## 实现步骤

### 步骤 1：创建实体类

1. **更新 `Article` 实体** (`src/modules/article/entities/article.entity.ts`)
   - 添加 `view_count`, `like_count`, `total_view_duration` 字段
   - 添加 `@Expose()` 装饰器

2. **创建 `ArticleView` 实体** (`src/modules/article/entities/article-view.entity.ts`)
   - 定义所有字段
   - 添加与 Article 的关联关系

3. **创建 `ArticleLike` 实体** (`src/modules/article/entities/article-like.entity.ts`)
   - 定义所有字段
   - 添加与 Article 的关联关系

### 步骤 2：创建 DTO

1. **创建 `ArticleStatsDto`** (`src/modules/article/dto/article-stats.dto.ts`)
   - 记录浏览时长的 DTO

### 步骤 3：更新 Service

1. **更新 `ArticleService** (`src/modules/article/article.service.ts`)
   - 添加 `incrementViewCount(articleId, fingerprint)` - 增加浏览次数（去重）
   - 添加 `recordViewDuration(articleId, fingerprint, duration)` - 记录浏览时长
   - 添加 `toggleLike(articleId, fingerprint, userId)` - 切换点赞状态
   - 添加 `getArticleStats(articleId)` - 获取文章统计信息

2. **创建 `ArticleStatsService`** (`src/modules/article/article-stats.service.ts`)
   - 专门处理统计相关的逻辑
   - 生成/验证访客指纹
   - Cookie 管理

### 步骤 4：更新 Controller

1. **更新 `ArticleController** (`src/modules/article/article.controller.ts`)
   - `GET /api/article/:id` - 获取文章详情时自动增加浏览次数
   - `POST /api/article/:id/view` - 记录浏览时长
   - `POST /api/article/:id/like` - 点赞/取消点赞
   - `GET /api/article/:id/stats` - 获取文章统计

### 步骤 5：访客指纹实现

1. **Cookie 指纹生成**
   - 使用 crypto 生成唯一指纹
   - 存储在 Cookie 中，名称：`visitor_fingerprint`
   - 过期时间：1小时
   - HttpOnly: true, Secure: 生产环境

2. **拦截器/中间件**
   - 创建 `VisitorFingerprintInterceptor`
   - 自动检查 Cookie，不存在则生成新的
   - 将指纹存储到 request 对象中供后续使用

### 步骤 6：创建数据库迁移

1. **生成迁移文件**
   - 使用 TypeORM 生成迁移
   - 包含新增的字段和表

2. **执行迁移**
   - 运行 `npm run migration:run`

## 防刷新机制

### 浏览人数防刷新
- 同一指纹在1小时内多次访问同一文章只计数一次
- 通过 `ArticleView` 表记录已访问的文章

### 点赞防重复
- 同一指纹或用户只能点赞一次
- 再次点击则取消点赞

## 浏览时长统计

### 实现方式
- 前端定时（每30秒）发送心跳包上报当前浏览时长
- 或前端在页面离开时上报总浏览时长
- 服务端累加时长到 `Article.total_view_duration`

## API 接口设计

### 1. 获取文章详情（自动增加浏览次数）
```
GET /api/article/:id
Response: { rlt: "0", msg: "success", data: Article }
```
- 自动检查指纹，未访问过则增加浏览次数

### 2. 记录浏览时长
```
POST /api/article/:id/view
Body: { duration: number }  # 本次浏览时长（秒）
Response: { rlt: "0", msg: "success" }
```

### 3. 点赞/取消点赞
```
POST /api/article/:id/like
Response: { rlt: "0", msg: "success", data: { liked: boolean, likeCount: number } }
```

### 4. 获取文章统计
```
GET /api/article/:id/stats
Response: {
  rlt: "0",
  msg: "success",
  data: {
    viewCount: number,
    likeCount: number,
    totalViewDuration: number,
    liked: boolean  # 当前用户是否已点赞
  }
}
```

## 注意事项

1. **Cookie 安全**
   - HttpOnly: true
   - SameSite: strict
   - Secure: 生产环境

2. **性能考虑**
   - 浏览次数统计可以考虑使用 Redis 缓存
   - 定期同步到数据库

3. **数据一致性**
   - 使用数据库事务确保统计数据一致性
   - 防止并发问题

4. **Swagger 文档**
   - 为新接口添加 Swagger 装饰器

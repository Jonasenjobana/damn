# 后端规范
## 数据库规范
1. 字段小写英文 + 下划线，不能有无意义命名
2. 统一公共字段`id`，`create_time`，`update_time`，`is_deleted`
3. 字段类型
    1. 数字类型
        - 主键 / 自增 ID：INT / BIGINT（大数据用 BIGINT）
        - 状态 / 枚举：TINYINT（0-255，替代字符串，性能更高）
        - 金额：DECIMAL(18,2)，禁止用 FLOAT/DOUBLE（精度丢失）
        - 年龄、排序：INT
    2. 字符串类型
        - 固定长度：CHAR（如手机号、身份证号）
        - 可变长度：VARCHAR（用户名、标题）
        - 长文本：TEXT（详情、备注），尽量少用，影响查询效率
    3. 时间类型
        - 创建 / 更新时间：DATETIME（推荐，带时区无关性）
        - 时间戳：TIMESTAMP（自动更新时用）
        - 禁止用字符串存储时间（无法排序、无法索引）
4. 字段约束设计（保证数据正确性）
- 非空约束 NOT NULL
    业务必填字段必须加非空，如用户名、手机号、状态
- 默认值 DEFAULT
    数字默认 0，字符串默认 ''，时间默认当前时间
    避免 NULL 查询、运算异常
- 主键约束
    每张表必须有主键，推荐自增数字 ID 或雪花 ID
- 唯一约束 UNIQUE
    手机号、用户名、订单号等唯一值必须加
- 外键慎用
    业务系统尽量不用物理外键，用逻辑外键，避免锁表、影响性能
5. 字段注释comment
6. 可以适度冗余，禁止大量冗余
    - 对于低频小字段不常修改
7. 索引设计
    - 高频查询字段必须建索引：用户 ID、订单号、手机号
    - 避免过度索引：索引会降低写入速度
    - 联合索引遵循最左前缀原则
    - 不索引低区分度字段：如性别、状态（只有 0/1）
